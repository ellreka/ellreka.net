## 背景

リモート会議では基本的に自分が話しているとき以外はミュートしておきたいのですが、話すたびにミュートボタンをクリックしなければならず面倒でした。

またキーボード等を操作しながらマイクのミュートを切り替えられないかと考えフットペダルで操作してみることにしてみました。

## 環境

- macOS Monterey 12.0.1

## 必要なもの

- フットペダル (私はこちらを購入しました)
  - https://www.amazon.co.jp/dp/B0759T7W7V
- BetterTouchTool
- Karabiner-Elements (必須ではない)
- Windows PC (フットペダルのキー設定を変更するのに必要です)

## 方法

まずマイクのミュート方法ですが、

1. OS側のマイク音量を操作する
2. アプリ側でマイクのミュートを切り替える

の二通りあります。

これは好みですが、両方試した結果私はアプリ側で制御することにしました。

理由としてはOS側で制御するとマイクがミュートされているのか分かりづらく不安になるからです。

また通話アプリをいくつも使っている場合だと①が良さそうですが、私はgoogle meetしか使っていないため②にしました。

### BetterTouchTool

BetterTouchToolにミュートを切り替えるためのアクションを追加します。

トリガーは後で好きなものを割り当てられるので今は何でも良いです。


#### ①OS側で制御する

アクションに「Apple Scriptを実行する(ブロック)」を登録します。

以下のコードを貼り付けます。(コードはこちらから拝借しました)
https://qiita.com/2357gi/items/0f69691ee9d7b2c8ebbe

```
set inputVolume to input volume of (get volume settings)
if inputVolume = 0 then
    set inputVolume to 100
else
    set inputVolume to 0
end if
set volume input volume inputVolume
return
```

これでトリガーを入力するたびマイクボリュームが100、0で切り替わるようになります。

もしペダルを踏んでいる間だけミュート解除したい場合は、ダウンとアップの２つのトリガーを登録する必要があります。

ダウン

```
set volume input volume 100
```

アップ

```
set volume input volume 0
```

#### ②アプリ側で制御する

アクションに「特定のアプリにショートカットを送信する」を登録します。

アプリを選択から任意のアプリを選択します。

meetの場合アプリが無いので meet.google.com にアクセスしPWAアプリをインストールしてください。

あとはショートカットの記録にミュートのショートカットキーを登録します。

```
meet: Command+D
zoom: Ctrl+Shift+M
```

これでアプリを開いた状態でトリガーを入力すればミュートが切り替わると思います。
